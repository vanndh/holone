# вклад в holone

спасибо, что помогаешь защитникам! самый ценный вклад — это **новые правила
детекта** и **свежие индикаторы компрометации (ioc)**, и для этого не нужно
трогать go.

## добавить правило детекта

правь [`rules/rules.json`](rules/rules.json). каждое правило — json-объект:

```json
{
  "id": "dl-certutil",
  "category": "download-exec",
  "severity": "high",
  "pattern": "(?i)\\bcertutil\\b[^\\n]*\\b(-?urlcache|/urlcache|-split|/split)\\b",
  "description": "certutil download cradle (lolbin staging)"
}
```

- `pattern` — это регэксп **go re2** (без look-around / обратных ссылок). в json
  экранируй бэкслеши (`\\b`, `\\\\` для буквального бэкслеша).
- `severity`: `high` (загрузка+запуск, анти-форензика, перехват трафика, lolbin,
  ioc — режется в block-режиме), `medium` (двойного назначения: простая
  загрузка, настройка proxy/route — только alert), `low` (слабый сигнал).
- категории: `download-exec`, `obfuscation`, `persistence`, `network`,
  `anti-forensics`, `locale`.

**каждому правилу — тест.** положи вредоносный образец файлом в
`internal/inspect/testdata/corpus/malicious/` (должен давать ≥1 находку) и, если
уместно, безобидный контр-пример в `…/corpus/clean/` (должен давать **0**
находок). потом:

```sh
go test ./internal/inspect/
```

хорошее правило ловит **технику**, а не одну буквальную строку — подумай про
алиасы, регистр, рефлексию и обфускацию, которыми атакующий будет обходить.

## добавить ioc

правь [`rules/blocklist.json`](rules/blocklist.json) — добавь домен, ip, путь,
имя задачи планировщика, имя процесса или sha-256, и укажи источник в описании pr
(ссылку на разбор / сэмпл).

## изменения в коде

```sh
go build ./...        # должно собираться под linux, darwin, windows
go vet ./...
go test ./...         # все тесты зелёные
gofmt -l .            # ничего не должно печатать
```

держи изменения сфокусированными, повторяй окружающий стиль и добавляй тест на
любое новое поведение. логика детекта — в `internal/inspect`, прокси — в
`internal/proxy`, ос-проверки — в `internal/sentinel`, сканер — в
`internal/scanner`.

## ложные срабатывания / пропуски

открой issue с точной входной строкой, что сработало (или нет) и что ожидалось.
файл-кейс в корпусе — самый быстрый воспроизводимый репро.

внося вклад, ты соглашаешься, что твоя работа лицензируется под mit-лицензией
проекта.
