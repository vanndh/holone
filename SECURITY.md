# политика безопасности

## область

holone — защитный инструмент. он снижает, но не убирает риск маршрутизации
llm-трафика через недоверенного провайдера. в частности, он **не** детектит
пассивный слив промптов; см. раздел readme «что честно не ловит».

## как сообщить об уязвимости

если нашёл способ **обойти детект** (вредоносный payload, который holone
пропускает) или изъян в самом прокси/сканере/sentinel — сообщи:

- через github security advisory (предпочтительно), либо
- обычным issue для низкорисковых пробелов детекта (например «правило x не ловит
  вариант y»).

для репортов про обход прикладывай точную строку payload и протокол
(anthropic / openai) — чтобы можно было превратить это в регрессионный тест в
корпусе.

пожалуйста, не используй holone, его правила или ioc-список для создания или
улучшения наступательного инструментария. эти данные существуют, чтобы защищать
пользователей.

## поддерживаемые версии

поддерживается последний релиз в основной ветке. правила и ioc обновляются
постоянно — подтягивай свежий `rules/`, чтобы оставаться в актуальном состоянии.
